# Arova Frontend GitHub Portfolio Readiness Audit

This document summarizes the repository hygiene audit for preparing the Arova Angular frontend application for a clean, public GitHub repository release.

---

## 1. Repository Structure Summary
Arova uses a modern Angular workspace containing standalone components, lazy-loaded feature modules, an E2E smoke test suite powered by Playwright, and a customized visual regression utility.

Key folders and configurations:
*   `src/`: Application codebase (assets, app shell, core guards/services, layouts, and feature routes).
*   `public/`: Global static files (including the high-fidelity cosmic `default-avatar.png`).
*   `e2e/`: Playwright end-to-end integration tests.
*   `scripts/visual-audit/`: Headless screenshot verification utility.
*   `design/`: Stitch redesign handoff design system assets.
*   `angular.json`, `playwright.config.ts`, `tsconfig.json`: Environment compilation and runtime settings.

---

## 2. Commit Policies & Safety Matrix

### Files Safe to Commit
*   All source codes under `src/` (HTML, TS, SCSS).
*   Test specifications under `e2e/` (e.g. `e2e/*.spec.ts`).
*   The screenshot audit scripts and guidelines (`scripts/visual-audit/`).
*   Global static assets (e.g., `public/assets/images/default-avatar.png`).
*   Clean, generalized project documents (e.g., `README.md`, `DESIGN_SYSTEM.md`, `STITCH_VISUAL_QA_REPORT.md`, `DOCS_INDEX.md`).
*   `package.json`, `package-lock.json`, `angular.json`, `tsconfig.json`, `playwright.config.ts`.

### Files / Folders That Must NOT Be Committed
*   `node_modules/`: Package dependencies (auto-installed on setup).
*   `dist/`: Compiled production bundles.
*   `.angular/`: Local compiler caches.
*   `visual-audit/latest/` and any generated screen captures (`*.png`, `*.jpg`, `*.webp`).
*   `playwright-report/` and `test-results/`: Local test execution reports.
*   Local application backup JSON files (`love-universe-backup-*.json`).
*   Local database files if any are generated by local tests or services (`*.db`, `*.sqlite`, `*.sqlite3`).
*   Local development environment files (`.env`, `.env.*`, `*.local`).

---

## 3. Secret & Privacy Risks Checked
1.  **Stripe/SMS Credentials**: Real Stripe client tokens, Apple/Google OAuth IDs, or Twilio phone gateways. Verified to contain simulated local mode fallback tags.
2.  **Emails & Phone Numbers**: Scanned for real developer email credentials or personal phone targets. Verified to be replaced with generalized placeholders (`owner@example.com`, `partner@example.com`).
3.  **Local File Paths**: Local absolute workspace folder paths (e.g., pointing to user directory structures) in documentation templates. Scanned and removed.
4.  **Cryptographic Keys**: Mock encryption keys are strictly defined as generic developer keys.
